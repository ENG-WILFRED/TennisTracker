# Activity System Database Integration - Complete

## ✅ Changes Made

### 1. **Database Schema** (`prisma/schema.prisma`)
Added new `Activity` model to store all activities:
```prisma
model Activity {
  id             String              @id @default(uuid())
  coachId        String              
  type           String              // 'session', 'tournament', 'restocking', 'player-reachout', 'email'
  date           String              // YYYY-MM-DD format
  startTime      String              // HH:MM format
  endTime        String              // HH:MM format
  title          String
  description    String?
  metadata       Json?               // Flexible storage for type-specific fields
  completed      Boolean             @default(false)
  createdAt      DateTime            @default(now())
  updatedAt      DateTime            @updatedAt
  
  coach          Staff               @relation("Activities", fields: [coachId], references: [userId], onDelete: Cascade)
  
  @@index([coachId])
  @@index([date])
  @@index([type])
  @@index([createdAt])
}
```

Also updated `Staff` model to include relationship:
```prisma
activities             Activity[]       @relation("Activities")
```

## 🔄 How It Works Now

### Creating an Activity
1. Coach clicks a day on calendar or "+ New Activity" button
2. Modal opens with dropdown of available courts
3. Coach selects court or "Other..." for custom location
4. Fills remaining fields
5. Clicks "Save Activity"
6. **NEW**: Activity is now saved to database
7. Returns activity with ID and timestamp
8. CalendarView refreshes sessions

### Retrieving Activities
```bash
GET /api/coaches/activities?coachId=<id>
GET /api/coaches/activities?coachId=<id>&date=2026-04-02
```

## 🛠️ Technical Details

### Flexible Metadata Storage
All activity type-specific fields are stored in `metadata` JSON field:
- Avoids creating separate tables for each type
- Allows easy addition of new field types
- Maintains backward compatibility
- Query-able with PostgreSQL JSON functions

### Court Dropdown Implementation
- API endpoint `/api/coaches/courts` fetches courts for coach's organization
- Dropdown shows: `Court Name (Surface, Indoor/Outdoor)`
- Default value when editing shows selected court
- "Other..." option shows custom input field

### API Responses
All activities properly stored with:
- UUID for ID (generated by Prisma)
- Timestamps (createdAt, updatedAt)
- Coach relationship for data integrity
- Metadata preserved as JSON
